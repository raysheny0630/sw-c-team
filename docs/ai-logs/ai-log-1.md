**프롬프트 (입력한 내용 요약)**

기획서 파일과 M1 기획서 가이드 파일을 첨부한 뒤, 가이드 양식에 맞춰 기획서를 GitHub 마크다운 업로드용으로 정제해달라고 요청함.

**AI 응답 요약**

기획서 내용을 바탕으로 `README.md`, `docs/M1-기획서.md`, `docs/requirements.md`, `docs/wbs.md` 4개의 마크다운 파일을 생성해주었다.

**비판적 검토**

- 제출 방식이 이러닝 PDF 업로드로 처음에 잘못 설정되어 있었다. 변경된 지침(GitHub 전체 관리)이 반영되지 않은 상태였다.
- 팀 구성 표에 학번이 포함되어 있었다. 공개 레포지토리에는 개인 정보를 올리면 안 되므로 문제가 있었다.
- FP 산정 시 ILF 항목과 요구사항 정의서 간 연결 근거가 부족했다.
- FR-04(유사 기사 리스트)가 프로젝트 핵심 차별점임에도 우선순위가 '하'로 설정되어 개요 내용과 모순이 있었다.

**팀/개인 수정 내용**

- PDF 제출 관련 내용을 전부 GitHub 마크다운 관리로 변경하였다.
- 팀 구성 표에서 학번 컬럼을 제거하고, 공개 레포 주의 문구를 추가하였다.
- `requirements.md`에 ILF 근거 항목(데이터 관리 항목 표)을 추가하여 FP 산정과 요구사항 간 논리적 연결을 확보하였다.
- FR-04 우선순위를 '하'에서 '중'으로 상향 조정하였다.

**최종 활용 결과**

GitHub 레포지토리(`sw-c-team`) 업로드용 마크다운 파일 4종 완성. `docs/` 폴더 구조에 맞게 정리하였다.
